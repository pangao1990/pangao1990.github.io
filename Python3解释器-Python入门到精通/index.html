Python3解释器-Python入门到精通

本文最后更新于：星期日, 五月 31日 2020, 2:37 下午

还记得在前面的课程中我们介绍过，在计算机和 Python程序之间有一个运行环境，这个运行环境就是我们今天要介绍的Python解释器。顾名思义啊，解释器的意思就是把人写出来的Python语言解释给计算机看，让计算机可以识别。

我们知道，整个 Python 语言从规范到解释器都是开源的。

开源，不是开源节流的意思，而是开放源码的意思。

所以理论上，只要水平够高，任何人都可以编写 Python 解释器来执行 Python 代码。

当然了，这个难度是很大的，我们至少得先学会使用解释器吧。

事实上，确实存在多种 Python 解释器。有CPython 、IPython、PyPy 等等。当我们从 Python 官方网站下载并安装好 Python 3 后，我们就直接获得了一个官方版本的解释器：CPython。这个解释器是用 C 语言开发的，所以叫 CPython。她是使用最广的 Python 解释器。本课程所有代码也都在 CPython下执行，所以其他的解释器我就不一一介绍了。

有两种方式启动 Python 解释器来运行脚本，一种是交互式编程，另一种是脚本式编程。

交互式编程

交互式编程的优点是方便快捷，不需要额外的编辑器辅助。

缺点就是没法保存代码，下次要使用还得重新敲代码。

脚本式编程

脚本式编程的优点是可以将代码保存为文本，使用时候直接调用就可以了。

缺点的话，我个人觉得没有缺点。所以我经常使用脚本式编程。
